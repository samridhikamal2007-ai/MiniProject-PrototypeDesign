# LearnSphere — CodeAlpha UI/UX Design Internship · Mini Project

A clickable prototype of an **Online Learning Platform** ("LearnSphere") plus a
**3-page design explanation document**, created for the CodeAlpha UI/UX Design
internship Mini Project (Prototype Design).

## 🔗 Live Prototype URL

The prototype is deployed and viewable in any browser:

**👉 https://sites.super.myninja.ai/69230a62-f760-45eb-b39f-1fa32c4d376b/2e3c3e25/index.html**

Every button, card, and tab navigates to the next screen — click through the
full user journey just like a Figma/Adobe XD prototype.

## 📱 The 6 Screens

1. **Onboarding** — `index.html` — welcome, value prop, get started
2. **Home / Discovery** — `home.html` — continue-learning hero, categories, course grid
3. **Course Detail** — `course-detail.html` — video preview, tabs, curriculum, sticky enrol CTA
4. **Video Lesson** — `lesson.html` — dark video player, controls, up-next list
5. **My Learning Dashboard** — `dashboard.html` — stats, weekly-goal ring, progress, badges
6. **Profile / Settings** — `profile.html` — identity, certificates, settings menu

Each screen also includes an on-page **screen navigator** at the bottom so
reviewers can jump to any screen instantly.

## 🎨 Design System at a glance

- **Primary:** Indigo `#5B5BF5` · **Accent:** Coral `#FF8A5C`
- **Type:** Inter (display 900 → caption 600)
- **Spacing:** 4pt scale · **Radius:** 10–30 + pill · **Tap targets:** ≥ 44pt
- **Accessibility:** WCAG AA contrast, colour + label states

## 🧠 Methodology

Built on the 5-stage design-thinking framework (Empathise → Define → Ideate →
Prototype → Test), informed by 2 personas and a competitive scan of
Udemy / Coursera / Skillshare. Usability decisions reference Jakob's, Hick's,
and Fitts's laws plus visibility-of-progress heuristics.

Full rationale is in the 3-page design explanation PDF.
